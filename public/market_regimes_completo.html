Change Point Detection — Más Allá del CUSUM Básico

El problema central de la detección de regímenes: dado un proceso estocástico {Xₜ}, determinar si sus parámetros fundamentales (media, varianza, covarianza) cambian en algún punto τ desconocido. Esta sección cubre los métodos modernos con complejidad O(n) que escalan a millones de observaciones.

▸
PELT — Pruned Exact Linear Time (Killick, Fearnhead, Eckley 2012)
core▾

PELT es el algoritmo de referencia para detección de múltiples change points cuando se necesita exactitud. La innovación central es usar programación dinámica con podas (pruning) que reducen la complejidad de O(n²) a O(n) en promedio.

1.1La mecánica de la poda — por qué funciona

El algoritmo minimiza una función de costo penalizada: Q(τ₁...τₖ) = Σᵢ C(y_{τᵢ₋₁:τᵢ}) + β·K, donde C(y_{s:t}) mide la "inconsistencia" del segmento [s,t] y β es el factor de penalización.

La poda funciona así: para cada punto t, se mantiene una lista de "candidatos". Si el costo de pasar por un candidato es mayor que el costo de saltarlo, ese candidato se poda permanentemente. En la práctica, esto elimina ~90% de los candidatos en datos financieros.

1.2La penalización óptima — el problema de β

La elección de β no es trivial:

Para detección sensible (trading): usar segmentación cuadrática con β = log(log(n))·2σ², lo que da ~20% más de change points.

Para detección robusta (análisis): usar BIC con validación fuera de muestra.

// PELT Cost Function C(y_{s:t}) = Σi=st-1 (yᵢ - μs:t)² // PELT recursion: F(t) = min_{s BIC: pen = log(n) · 2 · σ²_returns AIC: pen = 2 · 2 · σ²_returns // Para returns diarios típicos: pen ≈ 0.03-0.08
python
import numpy as np import ruptures as rpt returns = np.load('nq_daily_returns.npy') # PELT — detección automática del número de cambios algo_pelt = rpt.Pelt(model='rbf').fit(returns) changepoints_pelt = algo_pelt.predict(pen=0.05) # BinSeg — cuando necesitas especificar el número máximo algo_binseg = rpt.BinSeg(model='l2').fit(returns) changepoints_binseg = algo_binseg.predict(n_bkps=8) def compute_detection_metrics(true_breaks, detected_breaks, tolerance=5): hits = 0; lags = [] for tb in true_breaks: distances = [abs(tb - db) for db in detected_breaks] min_dist = min(distances) if distances else 999 if min_dist <= tolerance: hits += 1; lags.append(min_dist) precision = hits / len(detected_breaks) if detected_breaks else 0 recall = hits / len(true_breaks) if true_breaks else 0 adl = np.mean(lags) if lags else 999 return {'precision': precision, 'recall': recall, 'adl': adl}

BOCPD — Bayesian Online Changepoint Detection
avanzado▾

Adams & MacKay (2007) desarrollaron BOCPD, que calcula recursivamente P(rₜ | x₁:t) donde rₜ es el "run length" — el tiempo transcurrido desde el último change point.

2.1El concepto de run length

rₜ ∈ {0, 1, 2, ...} donde rₜ = k significa "el último cambio ocurrió hace exactamente k barras". La distribución P(rₜ | x₁:t) es el output principal del algoritmo.

2.2La hazard function — controlando la tasa de cambios

La hazard function H(r) define la probabilidad de un change point en cada run length r. Para datos de mercado, p_old = 0.999 (media de duración ≈ 1000 barras) es un buen punto de partida.

// BOCPD Recursion P(rₜ = r | x₁:t₋₁) = P(rₜ₋₁ = r-1 | x₁:t₋₁) · (1 - H(r)) + Σk>r P(rₜ₋₁ = k | x₁:t₋₁) · H(k) // Predictive probability (conjugate Normal-Inverse-Gamma) p(xₜ | r, params) = Student_t(xₜ; μ, σ²·(r+1)/r, ν)
python
import numpy as np from scipy.stats import t as student_t class BOCPD: def __init__(self, hazard_func=lambda r: 1.0/100): self.hazard = hazard_func self.run_lengths = np.array([1.0]) self.stats = [{'mu': 0.0, 'kappa': 1.0, 'alpha': 1.0, 'beta': 1.0}] def predictive_pdf(self, x, stats): df = 2 * stats['alpha'] scale = np.sqrt(stats['beta'] / stats['alpha'] / stats['kappa']) return student_t.pdf(x, df, loc=stats['mu'], scale=scale) def update(self, x): self.run_lengths = np.concatenate([[0.0], self.run_lengths]) self.stats = [{'mu': 0.0, 'kappa': 1.0, 'alpha': 1.0, 'beta': 1.0}] + self.stats new_run_lengths = np.zeros_like(self.run_lengths) for r in range(len(self.run_lengths)): H_r = self.hazard(r) prob_continue = (1 - H_r) * self.run_lengths[r] evidence = self.predictive_pdf(x, self.stats[r]) new_run_lengths[r] = prob_continue * evidence self.run_lengths = new_run_lengths self.run_lengths /= (self.run_lengths.sum() + 1e-100) return self.run_lengths.copy() def changepoint_probability(self): return self.run_lengths[0]
✦Aplicación directa para trading: BOCPD produce una serie temporal de P(r₀=0 | x₁:t). Cuando esta probabilidad supera un threshold (típicamente 0.6-0.8), tienes una señal de transición de régimen.
Adams & MacKay 2007bayesian_changepoint_detection

Structural Breaks — Tests Formales (Chow, Andrews, Bai-Perron)
core▾

Los tests de structural break responden a una pregunta diferente: dado que sabes (o sospechas) que hay un cambio en los parámetros, ¿cuál es la evidencia estadística para rechazarlo?

3.1Chow Test (1960) — el test clásico para un break point conocido

Gregory Chow desarrolló el test para cuando el break point τ es conocido a priori. La hipótesis nula es que los coeficientes son iguales antes y después de τ.

// Chow Test Statistic F = [(RSS_pooled - (RSS₁ + RSS₂)) / k] / [(RSS₁ + RSS₂) / (n - 2k)] // Crítico: τ debe ser conocido, no estimado de los datos

Limitación severa: si estimas τ de los datos, la distribución del estadístico cambia y el test se invalida.

3.2Andrews SupF y Bai-Perron — múltiples breaks desconocidos

Andrews (1993) propuso buscar el break point sobre todo el rango posible. Bai & Perron (1998, 2003) desarrollaron el framework completo para múltiples cambios desconocidos simultáneos. Este es el estándar dorado.

python
# Bai-Perron con ruptures import ruptures as rpt y = np.load('market_returns.npy') algo = rpt.Dynp(model="l2", min_size=5).fit(y) for k in range(1, 8): bkps = algo.predict(n_bkps=k) # Calcular BIC/AIC # CUSUM test cusum_stat = np.cumsum(y - y.mean()) cusum_scaled = cusum_stat / np.std(y)
⚠Problema de specification: Si el número de breaks m es seleccionado por los datos y luego se reportan intervalos de confianza, estos intervalos son inválidos.

Criterios de información y selección del número de regímenes K
core▾

La selección de K (número de regímenes) es uno de los problemas más delicados en la práctica. K=2 es casi siempre demasiado simple para mercados financieros; K>5 introduce sobreparametrización.

4.1Los tres criterios principales y sus propiedades

AIC (Akaike Information Criterion): AIC = 2k - 2ln(L̂). Tiende a sobreestimar K en muestras finitas. Usar cuando el objetivo es forecasting puro.

BIC (Bayesian Information Criterion): BIC = k·ln(n) - 2ln(L̂). Penaliza más agresivamente. Teóricamente consistente.

HQIC (Hannan-Quinn): HQ = 2k·ln(ln(n)) - 2ln(L̂). Compromiso entre AIC y BIC.

CriterioPenalizaciónTendenciaUso óptimo
AIC2kSobrestima KForecasting puro
BICk·ln(n)Subrestima KIdentificación de regímenes
HQIC2k·ln(ln(n))IntermedioDependencia temporal fuerte
4.2Cross-validation para series temporales

Para datos temporales, usar walk-forward validation:

python
def wf_cv_hmm(returns, max_K=6, train_size=500, step=50): results = {k: [] for k in range(1, max_K+1)} for start in range(0, len(returns)-train_size, step): train = returns[start:start+train_size] test = returns[start+train_size:start+train_size+step] for K in range(1, max_K+1): model = GaussianHMM(n_components=K) model.fit(train.reshape(-1,1)) ll = model.score(test.reshape(-1,1)) results[K].append(ll) return {k: np.mean(v) for k, v in results.items()}
◈Regla práctica para K en futuros: Para ES/NQ con datos diarios, K=3 captura "bull trending", "bear trending", y "range/mean-reversion". K=4 añade "crisis/crash" como estado separado.

Hamilton Markov-Switching Model — Arquitectura Completa

El paper de James D. Hamilton (1989, Econometrica) estableció el marco teórico que subyace a toda la detección moderna de regímenes.

▸
Especificación del MS-AR(p) y las tres variantes
avanzado▾

El modelo MS-AR(p): yₜ = μ(Sₜ) + Σᵢ φᵢ(Sₜ)(yₜ₋ᵢ - μ(Sₜ₋ᵢ)) + σ(Sₜ)εₜ

1.1MSM, MSV y MSMV — tres niveles de switch

MSM (Markov Switching in Mean): Solo la media μ(Sₜ) cambia.

MSV (Markov Switching in Variance): Solo la varianza σ²(Sₜ) cambia.

MSMV (Markov Switching in Mean and Variance): Ambos parámetros cambian. La especificación más general.

VarianteParámetrosAplicación
MSMμ₁, μ₂, σ²Cambios en drift
MSVμ, σ₁², σ₂²Regímenes de volatilidad
MSMVμ₁, μ₂, σ₁², σ₂²Regímenes completos
1.2La matriz de transición P y sus propiedades

La duración esperada en cada régimen: E[duración | régimen i] = 1 / (1 - pᵢᵢ)

Si p₁₁ = 0.97 (bull market), la duración esperada es 33 días.

// Matriz de transición K=2 P = | p₁₁ p₁₂ | | p₂₁ p₂₂ | E[duración | régimen i] = 1 / (1 - pᵢᵢ) // Ejemplo: p₁₁ = 0.97 → E[duración bull] = 33.3 días
python
from statsmodels.tsa.regime_switching.markov_autoregression import MarkovAutoregression model = MarkovAutoregression( endog=returns, k_regimes=2, order=1, switching_ar=True, switching_mean=True, switching_variance=True ) result = model.fit(method='bfgs', maxiter=500, n_init=50) filtered_probs = result.smoothed_marginal_probabilities[0]

Filtro de Hamilton y Smoother de Kim
avanzado▾

El filtro de Hamilton es el corazón computacional del MS model.

2.1El algoritmo de filtrado — paso a paso

Prediction step: ξₜ₊₁|t = P' · ξₜ|t

Update step: ξₜ₊₁|t₊₁ = (ηₜ₊₁ ⊙ ξₜ₊₁|t) / Σⱼ ηₜ₊₁(j)·ξₜ₊₁|j

// FILTRO DE HAMILTON // PREDICTION: t → t+1 ξₜ₊₁|t = P' · ξₜ|t // UPDATE: incorporar observación yₜ₊₁ ηₜ₊₁(i) = p(yₜ₊₁ | Sₜ₊₁=i, y₁:ₜ) ξₜ₊₁|t₊₁ = (ηₜ₊₁ ⊙ ξₜ₊₁|t) / Σⱼ ηₜ₊₁(j)·ξₜ₊₁|j Filtrada: ξₜ|t (datos hasta t) — online Suavizada: ξₜ|T (datos hasta T) — offline
python
import numpy as np from scipy.stats import norm class HamiltonFilter: def filter(self, y_series): T = len(y_series) filtered = np.zeros((T, self.K)) xi = np.array([0.7, 0.3]) for t in range(T): xi_pred = self.P.T @ xi eta = np.array([self.likelihood(y_series[t], s) for s in range(self.K)]) xi_unnorm = eta * xi_pred xi = xi_unnorm / xi_unnorm.sum() filtered[t] = xi return filtered
◈Precision numérica: Cuando las likelihoods son muy pequeñas, usar log-space para evitar underflow.
Hamilton 1989 JoEKim 1994 JoE

Hidden Markov Models — Arquitectura y los Tres Problemas de Rabiner

Lawrence Rabiner (1989, IEEE) formalizó la teoría de HMMs con tres problemas fundamentales.

▸
Problema 1 — Evaluation: Algoritmo Forward
core▾

Dada una secuencia de observaciones O y un modelo λ, ¿cuál es la probabilidad P(O|λ)?

1.1El algoritmo forward paso a paso

Definimos αₜ(i) = P(o₁, o₂, ..., oₜ, Sₜ=i | λ)

Inicialización: α₁(i) = πᵢ · bᵢ(o₁)

Recursión: αₜ(j) = bⱼ(oₜ) · Σᵢ [αₜ₋₁(i) · aᵢⱼ]

Terminación: P(O|λ) = Σᵢ αT(i)

// Algoritmo Forward // Inicialización (t=1) α₁(i) = πᵢ · bᵢ(o₁) // Recursión (t = 2, 3, ..., T) αₜ(j) = bⱼ(oₜ) · Σᵢ [αₜ₋₁(i) · aᵢⱼ] log αₜ(j) = log bⱼ(oₜ) + logsumexp( log αₜ₋₁(i) + log aᵢⱼ )
python
import numpy as np from scipy.special import logsumexp def forward_log(observations, A, B, pi): T = len(observations); K = len(pi) log_alpha = np.zeros((T, K)) log_alpha[0] = pi + B[:, observations[0]] for t in range(1, T): for j in range(K): log_alpha[t, j] = B[j, observations[t]] + logsumexp(log_alpha[t-1] + A[:, j]) log_likelihood = logsumexp(log_alpha[-1]) return log_alpha, log_likelihood
◈Importancia del log-space: Para secuencias largas (T > 100), el producto de probabilidades causa underflow. En log-space es estable.
▸
Problema 2 — Decoding: Algoritmo de Viterbi
core▾

¿Cuál es la secuencia de estados más probable que generó la secuencia de observaciones?

2.1Viterbi vs. decoding marginal

Viterbi encuentra Q* = argmax_Q P(Q, O|λ). El decoding marginal encuentra argmax_i P(Sₜ=i | O) independientemente.

Para trading, el Viterbi es más apropiado porque queremos la secuencia completa de regímenes.

python
def viterbi(observations, A, B, pi): T = len(observations); K = len(pi) delta = np.zeros((T, K)); psi = np.zeros((T, K), dtype=int) delta[0] = pi + B[:, observations[0]] for t in range(1, T): for j in range(K): scores = delta[t-1] + A[:, j] psi[t, j] = np.argmax(scores) delta[t, j] = np.max(scores) + B[j, observations[t]] q_star = np.zeros(T, dtype=int) q_star[T-1] = np.argmax(delta[T-1]) for t in range(T-2, -1, -1): q_star[t] = psi[t+1, q_star[t+1return q_star

Problema 3 — Learning: Baum-Welch (EM para HMM)
avanzado▾

Baum-Welch es el algoritmo EM para estimar los parámetros de un HMM.

3.1E-step: forward-backward

γₜ(i) = P(Sₜ=i | O, λ) — probabilidad marginal

ξₜ(i,j) = P(Sₜ=i, Sₜ₊₁=j | O, λ) — probabilidad conjunta

3.2M-step: actualización de parámetros

π̂ᵢ = γ₁(i)

âᵢⱼ = Σₜ ξₜ(i,j) / Σₜ Σₖ ξₜ(i,k)

μ̂ᵢ = Σₜ γₜ(i)·oₜ / Σₜ γₜ(i)

// Baum-Welch: E-step y M-step γₜ(i) = αₜ(i)βₜ(i) / Σⱼ αₜ(j)βₜ(j) ξₜ(i,j) = αₜ(i)aᵢⱼbⱼ(oₜ₊₁)βₜ₊₁(j) / P(O|λ) π̂ᵢ = γ₁(i) âᵢⱼ = Σₜ ξₜ(i,j) / Σₜ Σₖ ξₜ(i,k)
Rabiner 1989 IEEEBaum et al. 1970

Volatility Regimes — RS-GARCH, HAR-RV, VIX Term Structure

La volatilidad no es constante — cambia dramáticamente entre regímenes.

▸
HAR-RV — Heterogeneous Autoregressive Realized Volatility (Corsi 2009)
quant▾

HAR-RV modela la volatilidad realizada como combinación de componentes de diferentes horizontes temporales.

1.1La mecánica del HAR

RV_{t+1} = c + β_D·RV_t + β_W·RV^(W)_t + β_M·RV^(M)_t + εₜ

donde RV^(W)_t = media móvil 5 días, RV^(M)_t = media móvil 22 días.

// HAR-RV estándar RV_{t+1} = c + β_D·RV_t + β_W·RV^(W)_t + β_M·RV^(M)_t + εₜ β_D ≈ 0.4-0.6 (alta persistencia diaria) β_W ≈ 0.1-0.2 (contribución semanal) β_M ≈ 0.05-0.15 (contribución mensual)
python
import numpy as np def har_rv_regression(rv, horizons=[1, 5, 22]): df = pd.DataFrame({'RV': rv}) df['RV_D'] = rv for h, name in zip(horizons, ['RV_W', 'RV_M']): df[name] = rv.rolling(h).mean().shift(1) df = df.dropna() X = df[['RV_D', 'RV_W', 'RV_M']] y = df['RV'] return LinearRegression().fit(X, y)
▸
VIX Term Structure — Contango vs Backwardation
core▾

El VIX no es solo un número — su estructura temporal revela cómo el mercado pricea el riesgo.

2.1Contango — régimen normal de mercado

Contango: la curva de futuros del VIX tiene pendiente positiva. Los vendedores de opciones reciben prima por esperar.

2.2Backwardation — régimen de stress

Backwardation: VIX spot > VIX3M > VIX6M. Históricamente, backwardation >15% implica sharpe negativo para estrategias que short vol.

2.3VVIX — la volatilidad del VIX

VVIX alto + VIX bajo = complacencia frágil. El ratio VVIX/VIX > 5 sugiere fragilidad sistémica.

// VIX Term Structure CCI = (VIX_3M - VIX_1M) / VIX_1M FRAGILITY = VVIX / VIX // VVIX/VIX > 5 → régimen frágil

Microestructura y Order Flow — PIN, VPIN, OFI

La microestructura del mercado revela información que los precios agregados ocultan.

▸
PIN — Probability of Informed Trading (Easley & O'Hara)
avanzado▾

PIN cuantifica la probabilidad de que una orden provenga de un trader informado.

1.1El modelo de microestructura de PIN

PIN = λᵢ / (λᵢ + λᵤ). Un PIN > 0.5 indica mercado muy adversarial.

1.2VPIN — Volume-synchronized PIN

VPIN = |V_buy - V_sell| / V_total

VPIN > 0.5 → percentil 90 → régimen de alta toxicidad

VPIN > 0.7 → percentil 99 → crash inminente

// PIN PIN = α / (α + μ) // VPIN VPIN = |V_buy - V_sell| / V_total
python

Kelly Fraction por Régimen y Volatility Targeting
core▾

Kelly criterion es la apuesta óptima para maximizar el logarithm del capital.

1.1Kelly regime-conditional

f*(Sₜ=k) = μₖ / σₖ²

f_total = Σₖ P(Sₜ=k) · f*(k)

1.2Volatility Targeting

size = σ_target / σ_forecast(Sₜ)

Normal (σ=15%): size = 1x

Crisis (σ=45%): size = 0.33x

// Kelly Criterion f* = μ / σ² // Kelly regime-conditional f*(Sₜ=k) = μₖ / σₖ² f_total = Σₖ P(Sₜ=k | datos) · f*(k)
python
class RegimeAwareSizer: def kelly_by_regime(self, regime): params = self.regime_params[regime] f_raw = params['mu'] / (params['sigma']**2) return max(f_raw * 0.25, 0) def get_size(self, regime_probs): size = 0 for regime, prob in regime_probs.items(): size += prob * self.kelly_by_regime(regime) return size
✦Fractional Kelly: Usar ¼ Kelly para robustez porque la estimación de μ y σ tiene error.
